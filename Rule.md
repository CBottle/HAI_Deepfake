# HAI(하이)! - Hecto AI Challenge : 2025 하반기 헥토 채용 AI 경진대회

## 📋 대회 개요

### 배경
- **딥페이크 기술의 확산**: 생성형 AI 발전으로 실제 인물 얼굴을 정교하게 합성하는 딥페이크 기술이 빠르게 확산
- **양면성**: 콘텐츠 제작 등 긍정적 활용 증가 vs 허위 정보, 사생활 침해 등 부작용 심화
- **산업 수요**: 금융, 보안, 미디어 등 다양한 산업에서 실시간 탐지 기술 수요 급증
- **대회 연속성**: 상반기 중고차 이미지 차종 분류 대회에 이은 두 번째 대회

### 주제
**딥페이크 탐지 AI 모델 개발**

### 과제 설명
- **입력 데이터**: 이미지(jpg, jpeg, png, jfif) 또는 동영상(mp4, mov) → **반드시 이미지 단위로 입력**
- **출력**: 해당 콘텐츠가 진짜(Real)인지 딥페이크(Fake)인지 분류
- **특이사항**: 
  - 공식 학습 데이터 **제공되지 않음**
  - 추론을 위한 평가 데이터만 제공
  - 참가자가 규칙 범위 내에서 학습 데이터 직접 구축 필요

### 참가 자격
일반인, 학생 등 **누구나 참가 가능**

### 주최/주관
- **주최/주관**: 헥토(Hecto)
- **운영**: 데이콘

## 🏆 평가 방법

### 1. 리더보드 평가
- **평가 지표**: ROC-AUC (with Sample Weights)
  - 샘플별 가중치는 **공개하지 않음**
- **Public Score**: 전체 테스트 데이터 중 사전 샘플링된 50%
- **Private Score**: 전체 테스트 데이터 100%

### 2. 평가 방식
- **1차 평가**: 리더보드 Private Score 100%
- **2차 평가**: 코드 검증 통과한 최종 Private Score 상위 10팀 대상 **오프라인 발표 평가 진행**

### 3. 2차 평가 항목 (총 100점)

| 항목 | 심사 기준 | 점수 |
|------|-----------|------|
| **모델 성능** | 1) Private 리더보드 환산 점수 | 30 |
| **데이터 활용** | 2) 학습 데이터 수집 또는 생성 방식 | 10 |
| **모델 검증** | 1) 검증 데이터 구성 방식<br>2) 모델 검증 전략의 적절성 | 10 |
| **알고리즘** | 1) 데이터 전처리 방식<br>2) 모델 아키텍처 선정<br>3) 학습 및 추론 전략<br>4) 모델 성능 개선 방법 등 | 25 |
| **적용 가능성** | 1) 모델 학습 효율성<br>2) 모델 추론 속도<br>3) 실제 서비스 적용 시 활용성/확장성 등 | 20 |
| **전달력** | 1) 발표 시간 준수<br>2) 발표 자료 완성도와 발표 전달력<br>3) 질의 응답 완성도 | 5 |
| **총점** | | **100** |

## 📜 대회 규칙

### 1. 개인 또는 팀 참여 규칙
- **개인 참가**: 팀 신청 없이 제출탭에서 자유롭게 제출
- **팀 참가**: 팀 탭에서 가능 (팀 페이지에서 팀 병합 정책 확인)
- **팀 최대 인원**: 3명
- **중복 참가 금지**: 동일인이 개인 또는 복수 팀에 중복 등록 불가

### 2. 대회 핵심 규칙

#### (1) 이미지 단위 입력 처리 요건
- **영상 데이터**: 프레임 단위로 분해하여 각 프레임을 개별 이미지로 모델 입력
- **불허**: 복수 프레임을 동시에 모델 입력으로 사용하여 시간적 정보를 직접 활용하는 방식
- **허용**: 프레임별로 독립적으로 수행된 추론 결과를 종합하여 최종 결과 산출하는 후처리

#### (2) 사전 학습 모델 사용 가능 범위
- **기준 날짜**: 2025년 12월 29일 전(~2025.12.28)에 공식적으로 가중치가 공개된 모델만 사용 가능
- **라이센스**: 최소 비상업적 이용(NC)이 허용된 라이센스를 따라야 함
- **판단 기준**: '모델 아키텍처'가 따르는 라이센스 기준으로 판단
- **데이터셋 라이센스**: 사전 학습 과정에서 사용된 데이터셋의 라이센스 여부와 관계없이, 아키텍처 라이센스가 비상업적 이용조차 허용하지 않는 경우 사용 불가

#### (3) 외부 데이터 사용 가능
- **데이터 구축**: 공식 학습 데이터 미제공으로 참가자가 직접 수집/생성 가능
- **법적 준수**: 저작권·개인정보보호 등 관련 법령 준수, 법적 제약 없이 활용 가능한 데이터만 사용
- **API 활용**: 외부 데이터 수집·생성을 위한 API 활용 자유롭게 허용 (단, 모델 학습·추론에 직접 관여하지 않는 범위)
- **증빙 의무**: 외부 데이터 사용 시 출처와 생성 방식 명확히 기재 및 데이터 제출 필요

#### (4) API 사용 제한
- **불허**: 원격 서버를 통해서만 접근 가능한 API 형태의 모델/서비스(예: OpenAI API, Gemini API, 딥페이크 판별 서비스 API 등)를 모델 학습/추론 과정에 직접 사용
- **필수**: 모든 모델은 로컬 환경에서 직접 실행 가능해야 함
- **허용**: 외부 API를 활용한 '학습 데이터 수집·생성'은 허용 (단, 정적 형태로 저장 후 활용, 외부 데이터 사용 규칙 충족)

#### (5) 단일 모델 구성 요건
- **기본 원칙**: 추론은 반드시 단일 모델 사용
- **불허**: 복수 모델 조합, 서로 독립적으로 학습된 모델의 출력/임베딩 결합

**허용 범위:**
- 하나의 고정된 모델 아키텍처 기반, 단일 사전학습 모델 사용 추론
- 단일 백본(backbone)으로부터 추출된 feature 기반 head 확장 또는 auxiliary head 구성

**불허 사항:**
- 복수의 모델 또는 백본(backbone) 병렬 결합 구조
- 서로 독립적으로 학습된 모델의 예측 결과/임베딩 결합 방식
- 입력 특성에 따라 모델/백본(backbone) 선택·전환 구조
- Test-Time Augmentation(TTA) 등 반복 추론 후 결과 결합 방식

- 측정 방식: 동일 환경에서 운영진이 추론 코드 5회 실행하여 측정한 소요 시간의 평균값

### 코드 제출 요건

#### Docker 기반 재현 가능한 소스 패키지 제출
2차 평가 대상자는 주최측이 제시하는 **프로젝트 코드 구조(.py 기반)에 맞게 반드시 구성**

```
your_submission/                 # 필수요건 (제출 ZIP 최상위 디렉터리)
├── model/                       # 필수요건
│   ├── model.pt                 # 필수요건 – 최종 추론에 사용하는 단일 모델 weight 1개
│
├── src/                         # 희망요건 – 모델/데이터/유틸 모듈 분리
│   ├── models.py                # 희망요건 – 모델 정의
│   ├── dataset.py               # 희망요건 – 데이터 로더/전처리
│   └── utils.py                 # 희망요건 – 공통 유틸 함수
│
├── config/                      # 필수요건
│   ├── config.yaml              # 필수요건 – 하이퍼파라미터, 경로 키, 모델명 등
│
├── env/                         # 필수요건
│   ├── Dockerfile               # 필수요건 – 제출 Docker 이미지 재현용
│   ├── requirements.txt         # 필수요건 – 추가 Python 라이브러리 목록
│   └── environment.yml          # 희망요건 – 로컬/연구용 conda 환경 정의
│
├── train_data/                  # 필수요건
│   └── 학습 데이터              # 필수요건 – 학습 데이터(재현용, 데이터 전수), 출처 및 라이선스
├── test_data/                   # 필수요건
│   └── 평가 데이터              # 필수요건 – 경진대회 제공 평가 데이터셋
│
├── train.py                     # 필수요건 – config, train_data 기반 학습 코드
├── eval.py                      # 희망요건 – 내부 검증용 평가 코드(ROC-AUC 등)
├── inference.py                 # 필수요건 – 채점용 추론 엔트리 포인트
└── README.md                    # 필수요건 – 전체 구조/환경/실행 방법 설명
```
